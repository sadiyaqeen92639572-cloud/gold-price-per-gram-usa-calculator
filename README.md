# Gold Price Per Gram USA Calculator

Live gold price per gram calculator in USD — 24K, 22K, 18K, 14K, 10K and cash-for-gold, updated three times daily. US-market companion to [goldpricepergram.co.uk](https://goldpricepergram.co.uk).

**Site:** [usgoldpricepergram.com](https://usgoldpricepergram.com)

## Why a separate site instead of a `/us/` subfolder

`goldpricepergram.co.uk` is a `.co.uk` ccTLD — a strong UK-only geo signal to Google. A `/us/` subfolder under it would be unlikely to rank for native US searches regardless of content quality. This is a standalone `.com` site instead, with its own repo, domain, and purity set (US karats aren't a renaming of UK carats — 10K, the US low end, is a genuinely different purity from UK's 9ct, not the same thing under a different name).

## Data sourcing — reads the UK site's data, no goldapi.io call of its own

Both this site's `update-prices.yml` and the UK site's run on the same 3×/day cron. If this site called [goldapi.io](https://www.goldapi.io/) directly too, that would double the call volume against a single 100/month free-tier quota (~180/month combined) and risk a freemium multi-account ToS issue if a second account were used instead. So instead: this site fetches the UK site's already-published `gold-data.json` (`https://goldpricepergram.co.uk/gold-data.json` — a public static file, served like any other page by GitHub Pages, free and uncapped), then converts those GBP prices to USD using a live GBP→USD rate from [frankfurter.dev](https://frankfurter.dev/) (ECB rates, no API key, no request cap). **Zero additional goldapi.io calls.** This is disclosed on the `/methodology/` page rather than presented as if it were a native USD feed.

## Structure

- `index.html` — pillar calculator (all karats).
- `24k-gold-price-per-gram/`, `22k-gold-price-per-gram/`, `18k-gold-price-per-gram/`, `14k-gold-price-per-gram/`, `10k-gold-price-per-gram/` — one calculator per karat, each with a distinct US-market angle (investment coins, South Asian/Middle Eastern-American wedding jewelry, engagement rings, everyday US jewelry, US legal minimum), generated by `generate-pages.js`.
- `cash-for-gold-price-per-gram/` — the US market's dominant search term for scrap/selling (vs UK's "scrap gold").
- `methodology/` — data sources, FX-derivation formula, and a live worked example.
- `site.config.json` — single source of truth for karat fractions, the UK `gold-data.json` URL, the frankfurter.dev endpoint, buyer discount factor, and the page list.
- `update-data.js` — two-stage fetch: the UK site's published `gold-data.json` (GBP, no goldapi.io call), then frankfurter.dev FX rate, then derives USD prices. Tracks the two upstream calls independently (`lastUpdated` for the UK gold price, `fxLastUpdated` for the FX rate) so a failure in one doesn't silently misrepresent the freshness of the other — see `fallbackComponent` in `gold-data.json`.
- `generate-pages.js` — regenerates the 7 satellite pages. Also emits reciprocal `hreflang` tags (`en-US`/`en-GB`) on the pillar and the 4 karats that genuinely exist on both sites (24K/22K/18K/14K — not 10K, which has no UK equivalent).

## Free Companion Tools

### [US Gold Melt Value Calculator](https://sadiyaqeen92639572-cloud.github.io/us-gold-melt-value-calculator/)

Free tool to estimate what a gold item is worth from its weight and karat — accepts grams, pennyweight (dwt) or troy ounces, unlike most calculators that only take grams. Pulls a live per-gram price from this site's `gold-data.json` for the estimate. For the full live price broken out by karat (24K/22K/18K/14K/10K) and a cash-for-gold estimate, use the [main calculator](https://usgoldpricepergram.com) above.

## Cross-linking with the UK site

No automatic geo-redirect between the two domains (safer than cross-domain redirect logic). Instead, a static "🇺🇸 US site" / "🇬🇧 UK site" banner link on every page of both sites, plus hreflang where a genuine page-level equivalent exists.

## Local testing

```bash
node update-data.js --mock              # inject mock price + FX data without calling either real endpoint
node update-data.js --force-fail-gold   # test fallback when the UK gold-data.json fetch fails but FX succeeds
node update-data.js --force-fail-fx     # test fallback when FX fails but the UK gold-data.json fetch succeeds
```

## Deploy

GitHub Pages via `.github/workflows/pages.yml` (deploys on push to `main`, and also on completion of `update-prices.yml` via a `workflow_run` trigger — required because the default `GITHUB_TOKEN` used by `update-prices.yml`'s git push does not itself trigger other workflows). Prices are refreshed by `.github/workflows/update-prices.yml` (cron, 3×/day).

No API secret required — the only two upstream calls are a plain HTTPS GET of the UK site's public `gold-data.json` and a keyless frankfurter.dev request.
